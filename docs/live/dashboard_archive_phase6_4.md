# Live Phase 6.4 — Archive dashboard mode

Live Phase 6.4 adds a **summary-first dashboard mode** for long live-loop studies.  The older dashboard mode points at a single live run directory, such as `outputs/live_adaptation_replay`.  Archive mode points at a long-term archive root, such as `live_archive`, and is designed for days, weeks, or months of data.

This matters because a live loop can produce too much data for an interactive dashboard to load directly.  Archive mode does **not** open every raw partition by default.  It reads:

1. `manifest.csv`, which says what files exist and how large they are.
2. Compact summary tables from `dmdc archive-summarize`.
3. Quicklook image metadata from `dmdc archive-quicklook`.
4. Alert summaries, trust summaries, bias summaries, and residual summaries.

For the full data-management story, see:

- [`docs/live/archive_phase6_2.md`](archive_phase6_2.md) for archive layout and manifest design.
- [`docs/live/summaries_quicklooks_phase6_3.md`](summaries_quicklooks_phase6_3.md) for summaries and static quicklooks.
- [`docs/live/adaptation_phase6.md`](adaptation_phase6.md) for bias-correction records.
- [`docs/live/dashboard_phase5.md`](dashboard_phase5.md) for the single-run dashboard.

## Typical workflow

First create a live run with monitoring/adaptation:

```bash
dmdc live-replay-adapt --config configs/templates/live_replay_adapt.toml
```

Archive the run:

```bash
dmdc archive-run \
  --run-dir outputs/live_adaptation_replay \
  --archive-root live_archive \
  --format parquet
```

Build compact summaries:

```bash
dmdc archive-summarize \
  --archive-root live_archive \
  --windows-seconds 60 300 3600
```

Generate quicklook plots:

```bash
dmdc archive-quicklook \
  --archive-root live_archive \
  --window-label 60s
```

Open the archive dashboard:

```bash
dmdc live-dashboard \
  --archive-root live_archive \
  --mode archive \
  --window-label 60s
```

or use the config template:

```bash
dmdc live-dashboard --config configs/templates/live_dashboard_archive.toml
```

---

## What the archive dashboard shows

The archive dashboard includes tabs for:

- **Overview:** archive status, manifest size, archived rows, data kinds, alert counts, and trust summary.
- **Summary plots:** interactive trust, residual, bias, and state-summary plots.
- **Quicklooks:** static PNGs generated by `archive-quicklook` for fast browsing.
- **Manifest:** searchable table of partitioned archive files.
- **Alerts:** compact alert summary table and bar chart.
- **Raw summary tables:** direct access to the summary CSVs used by the dashboard.

The dashboard is read-only.  It does not modify the archive, update a model, or control hardware.

---

## Why archive mode is summary-first

For high-rate live data, the raw stream can become enormous.  A dashboard that tries to read every raw file will eventually become slow or unusable.  Archive mode follows this rule:

```text
summary tables first
quicklook plots second
manifest drill-down third
raw partition files only when needed
```

This makes it possible to inspect months of runs by first looking at small tables such as:

```text
live_archive/summaries/trust_summary_60s.csv
live_archive/summaries/residual_summary_60s.csv
live_archive/summaries/bias_summary_60s.csv
live_archive/summaries/alert_summary.csv
```

## Choosing the window label

The `--window-label` should match a summary window you already created.

For example, if you ran:

```bash
dmdc archive-summarize --archive-root live_archive --windows-seconds 60 300 3600
```

then valid dashboard window labels include:

```text
60s
300s
3600s
```

Use shorter windows for detailed debugging and longer windows for multi-day/month views.

## Smoke test without launching Streamlit

For CI or a terminal-only workstation, run:

```bash
dmdc live-dashboard \
  --archive-root live_archive \
  --mode archive \
  --write-summary-only
```

This writes:

```text
live_archive/archive_dashboard_summary.json
```

That JSON file is useful for automated checks and reports.

## Recommended demo sequence

When presenting the repo, this is the cleanest story:

```bash
# 1. Run live replay with monitoring and bias correction.
dmdc live-replay-adapt --config configs/templates/live_replay_adapt.toml

# 2. Archive the run.
dmdc archive-run --run-dir outputs/live_adaptation_replay --archive-root live_archive

# 3. Build summaries and quicklooks.
dmdc archive-summarize --archive-root live_archive --windows-seconds 60 300 3600
dmdc archive-quicklook --archive-root live_archive --window-label 60s

# 4. Open the dashboard.
dmdc live-dashboard --archive-root live_archive --mode archive --window-label 60s
```

This demonstrates the full chain:

```text
live data → estimation → monitoring → bias correction → archive → summaries → dashboard
```
